前言：本篇博客主要介绍kubernetes集群如何与ceph集群进行对接，将ceph作为kubernetes的后端存储实现pvc的动态供应。本文中的ceph和kubernetes为一套集群。

主机列表

K8s集群角色 ceph集群角色 IP 内核
master-1 mon、osd节点 172.16.200.101 4.4.247-1.el7.elrepo.x86_64
master-2 mon、osd节点 172.16.200.102 4.4.247-1.el7.elrepo.x86_64
master-3 mon、osd节点 172.16.200.103 4.4.247-1.el7.elrepo.x86_64
node-1 osd节点 172.16.200.104 4.4.247-1.el7.elrepo.x86_64

升级内核的原因：ceph官方推荐4.1.4版本以上的版本

对接前需要保证ceph集群已经处于健康状态

1root ~ >>> ceph health
2HEALTH_OK

对接过程

创建存储池mypool

1root ~ >>> ceph osd pool create mypool 128 128 #设置PG和PGP为128
2pool 'mypool' created
3
4root ~ >>> ceph osd pool ls
5mypool

创建secret对象

在ceph中创建kube用户并授予相应的权限

1root ~ >>> ceph auth get-or-create client.kube mon 'allow r' osd 'allow class-read object_prefix rbd_children,allow rwx pool=mypool'

 1root ~/k8s/ceph >>> cat ceph-secret.yaml 
 2apiVersion: v1
 3kind: Secret
 4metadata:
 5  name: ceph-kube-secret
 6  namespace: default
 7data: 
 8  key: "ceph auth get-key client.kube | base64" 
 9type: kubernetes.io/rbd
10---
11apiVersion: v1
12kind: Secret
13metadata:
14  name: ceph-admin-secret
15  namespace: default
16data:
17  key: "ceph auth get-key client.admin | base64"
18type: kubernetes.io/rbd

创建上述资源

1root ~/k8s/ceph >>> kubectl apply -f ceph-secret.yaml

创建StorageClass

 1root ~/k8s/ceph >>> cat ceph-storageclass.yaml
 2apiVersion: storage.k8s.io/v1
 3kind: StorageClass
 4metadata:
 5  name: ceph-storageclass
 6provisioner: kubernetes.io/rbd
 7parameters:
 8  monitors: 172.16.200.101:6789,172.16.200.102:6789,172.16.200.103:6789  #定义mon节点，ceph-mon默认监听6789端口
 9  adminId: admin
10  adminSecretName: ceph-admin-secret
11  adminSecretNamespace: default
12  pool: mypool                                                           #定义ceph存储池
13  userId: kube
14  userSecretName: ceph-kube-secret
15  userSecretNamespace: default
16  fsType: ext4
17  imageFormat: "2"
18  imageFeatures: "layering"

1root ~/k8s/ceph >>> kubectl apply -f ceph-storageclass.yaml
2root ~/k8s/ceph >>> kubectl get sc
3NAME                PROVISIONER    AGE
4ceph-storageclass   kubernetes.io/rb   87s

创建pvc

 1root ~/k8s/ceph >>> cat ceph-storageclass-pvc.yaml 
 2apiVersion: v1
 3kind: PersistentVolumeClaim
 4metadata:
 5  name: ceph-test-claim
 6spec:
 7  storageClassName: ceph-storageclass  #对应storageclass的名称
 8  accessModes:
 9    - ReadWriteOnce
10  resources:
11    requests:
12      storage: 5Gi                    #申请5G空间的存储空间
  • ReadWriteOnce: 读写权限，并且只能被单个Node挂载
  • ReadOnlyMany:只读权限，允许被多个Node挂载
  • ReadWriteMany:读写权限，允许被多个Node挂载

查看pod的状态，如果发现pod并未处于running状态并且事件中有如下的报错。并且k8s集群的各个节点确认已安装与ceph版本一致的ceph-common软件包，这时需要更新storageclass的Provisioner，用外部的提供者代替k8s自带的kubernetes.io/rbd以实现pvc的动态供应。

1rbd: create volume failed, err: failed to create rbd image: executable file not found in $PATH:

排障步骤

出现这个报错问题的原因其实很简单：gcr.io中自带的kube-controller-manager镜像没有自带rbd子命令。所以如果是镜像方式启动的kube-controller-manager就会遇到这个问题。

Github相关issue：Error creating rbd image: executable file not found in $PATH · Issue #38923 · kubernetes/kubernetes · GitHub

部署外部provisioner

定义外部的provisioner，由这个provisioner拿之前定义的秘钥创建ceph镜像。同时对外提供ceph.com/rbd的供应者。相关的定义如下：

 1root ~/k8s/ceph >>> cat storageclass-fix-deployment.yaml 
 2apiVersion: apps/v1
 3kind: Deployment
 4metadata:
 5  name: rbd-provisioner
 6  namespace: kube-system
 7spec:
 8  replicas: 1
 9  selector:
10    matchLabels:
11      app: rbd-provisioner
12  strategy:
13    type: Recreate
14  template:
15    metadata:
16      labels:
17        app: rbd-provisioner
18    spec:
19      containers:
20      - name: rbd-provisioner
21        image: "quay.io/external_storage/rbd-provisioner:latest"
22        env:
23        - name: PROVISIONER_NAME
24          value: ceph.com/rbd
25      serviceAccountName: persistent-volume-binder

这里必须明确指明ServiceAccountName为persistent-volume-binder，默认的default账户没有权限列出相应资源

创建该deployment资源

1root ~/k8s/ceph >>> kubectl apply -f storageclass-fix-deployment.yaml

更新原有storageclass

1...
2metadata:
3  name: ceph-storageclass
4#provisioner: kubernetes.io/rbd
5provisioner: ceph.com/rbd
6...

1root ~/k8s/ceph >>> kubectl get sc
2NAME                PROVISIONER    AGE
3ceph-storageclass   ceph.com/rbd   64m

重新创建pvc及pod

1root ~/k8s/ceph >>> kubectl delete -f ceph-storageclass-pvc.yaml && kubectl apply -f ceph-storageclass-pvc.yaml
2
3root ~/k8s/ceph >>> kubectl delete -f ceph-busybox-pod.yaml && kubectl apply -f ceph-busybox-pod.yaml

pod成功进入running状态

1root ~/k8s/ceph >>> kubectl get pods              
2NAME       READY   STATUS    RESTARTS   AGE
3ceph-pod   1/1     Running   0          47s
